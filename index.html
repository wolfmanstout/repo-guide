    repo-guide

    The repo-guide repository is a Python project that uses AI to generate documentation for code repositories. It leverages large language models (LLMs) to understand the structure and content of a repository and create Markdown guides. The generated documentation is then served using MkDocs.

    The project has the following dependencies:

    • bleach and bleach-allowlist: Used to sanitize HTML in the generated documentation.
    • click: Used to create a command-line interface for the repo-guide application.
    • gitpython: Used to interact with Git repositories to understand file history and identify changes.
    • llm and llm-gemini: Used to interface with LLMs, specifically Google's Gemini models.
    • mkdocs and mkdocs-material: Used to generate and serve the documentation website.
    • tqdm: Used to display progress bars during documentation generation.
    • tiktoken: Used to count tokens for language model input and output.
    • magika: (optional dependency) Used to classify file types and ignore binary files.

    The repository is structured as follows:

    • The tests directory contains a comprehensive suite of tests for the repo-guide application. It includes tests for prompt construction, handling of various file types, the construction of repository and file URLs, and end-to-end documentation generation. It uses fixtures to mock dependencies like the LLM and the repository itself.
    • The src directory contains the source code for the repo-guide application.
      • The src/repo_guide subdirectory contains the core logic of the application, including the command-line interface defined in cli.py. This file is responsible for parsing command-line arguments, creating a DocGenerator, and setting up the documentation server. The DocGenerator class handles loading existing documentation, filtering files, building prompts for the LLM, generating documentation, managing tokens, and writing the mkdocs configuration. The files __init__.py and __main__.py make the repo_guide directory a Python package and provide a way to invoke the CLI from the command line.
    • The _github directory contains configuration files for GitHub features.
      • The _github/workflows directory contains the GitHub Actions workflows for testing, publishing the package to PyPI, and publishing the documentation website. The workflows defined in test.yml, publish.yml, and publish_site.yml handle continuous integration, deployment to PyPI, and deployment of the documentation website to GitHub Pages.
    • The file pyproject.toml defines project metadata, dependencies, and build system settings. It also configures the ruff linter.
    • The file README.md provides an overview of the repo-guide project, installation instructions, usage examples, troubleshooting tips, and development guidelines.
    • The file LICENSE contains the Apache 2.0 license under which the project is distributed.
    • The file pre-commit-config.yaml configures the pre-commit hooks to enforce code quality using ruff.
    • The file gitignore specifies files and directories that should be ignored by Git.
    • The file .python-version specifies the Python version used by the project.
    • The file .gitattributes specifies how Git should handle line endings.

    The core logic of repo-guide is encapsulated in the DocGenerator class, which is instantiated and invoked by the command-line interface in cli.py. The DocGenerator class reads file contents and structure from a Git repository and uses a custom XML format to generate prompts for the LLM, which produces the documentation included in the generated README.md files. The mkdocs configuration generated by repo-guide uses the bleach library to sanitize HTML in the documentation.

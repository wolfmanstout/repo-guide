    The repo_guide/src/repo_guide directory contains the core logic of the repo-guide tool. It is a Python package, indicated by the presence of an __init__.py file.

    The __main__.py file is the entry point for the command-line interface (CLI). It imports the cli function from cli.py and executes it when the script is run directly. This makes the package executable from the command line.

    The cli.py file is the heart of the application, implementing the DocGenerator class which generates the documentation and the CLI using the click library. The cli function defines the command-line interface for repo-guide, including options for input and output directories, LLM model selection, token counting, and more. This file is responsible for:

    • Parsing command-line arguments.
    • Initializing the DocGenerator class.
    • Walking the input directory.
    • Prompting the language model.
    • Writing the generated documentation to the output directory.
    • Writing an mkdocs.yml configuration file.
    • Optionally starting a local documentation server.
    • Filtering files and directories using ignore patterns, git metadata, or magika if available.

    The DocGenerator class does the bulk of the work, orchestrating the documentation generation process:

    • The __init__ method initializes the DocGenerator class, sets up the repository object, resolves paths, and configures the language model. It also attempts to determine the repository's URL and file prefix for creating links in the generated documentation.
    • The _prompt_with_backoff method handles making requests to the LLM with exponential backoff in case of errors.
    • The _safe_read_file method safely reads file contents, handling different encodings.
    • The _forward_slash_path method ensures path separators are forward slashes, as required by the XML.
    • The _build_prompt method constructs the XML input for the LLM, including the current directory's path, subdirectories, and files. It also includes the content of existing README.md files to provide context. It truncates file contents if the --files-token-limit is set.
    • The _build_system_prompt method constructs the system prompt that instructs the LLM how to process the XML input. It includes custom instructions if provided.
    • The load_existing_docs method loads existing documentation from the filesystem if resuming is enabled.
    • The generate_docs method walks through the input directory, filtering out unwanted files and directories. For each directory, it calls the LLM with a generated prompt, and then saves the resulting output as a README.md file in the output directory.
    • The write_mkdocs_configuration writes the mkdocs.yml file for use by MkDocs. The on_page_content hook cleans the generated HTML by stripping any non-markdown tags and attributes.

    Together, these files provide the core functionality of repo-guide. The cli.py module is the interface between the user and the underlying logic of the DocGenerator class. The __main__.py file makes the cli.py module executable from the command line.
